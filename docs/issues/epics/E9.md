## Zielbild

Die HTTP-Schicht des ITS: zwei FastAPI-Router (Schüler- und Lehrerseite) plus ein gemeinsames Pydantic-Schema- und Fehlermodell. Die API ist eine **dünne, validierende Naht** ohne eigene Geschäftslogik. Jeder personenbezogene Zugriff läuft durch eine `scoped_session(principal)` (setzt PG-Rolle + Session-Variable → speist Row-Level Security, P1); Aggregate ausschliesslich durch die Min-Cohort-Schwelle (SAF-3). Safety-Ausnahmen werden zentral auf neutrale `403`/`404` gemappt — keine Detail-Leaks.

Die zentrale Trennung: die **schonende Schülersicht** (nur `mastery`) gegenüber der **vollständigen Lehrersicht** inkl. `uncertainty` (Open Learner Model, P5). Lehrer-Interventionen (Notiz + optionaler Mastery-Override) sind erstklassiger Pfad (P6), nicht Reporting.

## Tasks

- **API-3 — Pydantic-Schemas + Fehlermodell**: definiert Request/Response-Schemas (`TurnRequest`, `TurnResponse`, `SkillMastery`, `CohortStat`, …) und den zentralen Exception-Handler, der Safety-Ausnahmen neutral auf `403`/`404` mappt. Basis für beide Router.
- **API-1 — Student-Endpoints** (`safety-critical`): `POST /student/turn` fährt den LangGraph-Agent-Loop in einer `scoped_session`; `GET /student/mastery` liefert die schonende Sicht.
- **API-2 — Teacher-Endpoints** (`safety-critical`): RLS-gefiltertes Lesen des Open Learner Model inkl. `uncertainty`, Kohorten-Verteilung über die Min-Cohort-Schwelle und das Schreiben von Notizen/Overrides.
