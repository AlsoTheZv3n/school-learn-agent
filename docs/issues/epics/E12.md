## Zielbild

E12 etabliert die **Teststrategie als Querschnitt** über die gesamte ITS-Plattform und die **CI-Härtung**. Ziel: die Safety-Eigenschaften (Zeilenisolation via RLS, Min-Cohort) maschinell und **CI-blockierend** absichern (P1) und die Kernlogik (Lernmodell, Grading, Agent-Loop, API) entlang der Testpyramide abdecken. Am Ende läuft `uv run pytest` grün in CI, wobei ein **vorgelagerter, blockierender Safety-Schritt** den Build sofort abbricht, wenn die Isolationsgarantien reissen.

## Teststrategie (Pyramide)

| Ebene | Was | Gegen was |
|---|---|---|
| **Safety** (kritisch) | RLS-Isolation, Min-Cohort | echtes Postgres mit RLS |
| **Unit** | BKT, Grader, Parser, Anonymizer | reine Funktionen, keine DB |
| **Integration** | Agent-Loop (ein voller Turn), Tracing-Schreibpfad | Test-DB |
| **E2E-Smoke** | Login → Session → Antwort → Mastery; Lehrer sieht Stand | API (+ Frontend) |

Grundsätze: Safety-Tests **blockieren den Merge** (P1); Tests laufen gegen **echtes Postgres mit aktivierter RLS** (gegen SQLite würde die Zeilenisolation gar nicht geprüft); Unit-Tests reiner Logik brauchen **keine** DB und sind schnell. Alle Läufe über **`uv run`** — niemals `pip` (P9).

## Tasks (je ein Satz Zweck)

- **TST-1 — Pytest-Fixtures (DB pro Test, Rollen):** `tests/conftest.py` mit transaktionaler Test-DB-Fixture (Alembic `upgrade head` inkl. `rls.sql`) und `DBFactory` (`as_student/as_teacher/as_admin`, fail-closed) — das Fundament, auf dem Safety-Tests, TST-3 und TST-4 laufen.
- **TST-2 — Unit-Tests Lernmodell & Grading:** DB-freie, schnelle Tests für BKT (`test_bkt.py`), Math-Grader gegen kuratierten Key (`test_grading/`), Content-Parser und Anonymizer (P2/P4/P7 abgesichert).
- **TST-3 — Integrationstests Agent-Loop:** ein voller `ANSWER`-Turn end-to-end gegen die Test-DB beweist `confidence == 1.0` (kuratiert, P2) und aktualisierte Mastery (Modell, nicht Agent, P3).
- **TST-4 — E2E Smoke (API + Frontend):** HTTP-Smoke (zwingend) Login → Session → Antwort → Mastery; Lehrer sieht Stand inkl. Unsicherheit, Schüler nicht (Präsentations-Trennung, P5); Browser-E2E optional.

> Hinweis: Der **vorgelagerte, blockierende CI-Schritt** für `test_rls.py` + `test_cohort_threshold.py` (docs/10 §7) wird in diesem Epic verdrahtet und ist die organisatorische Klammer um SAF-4.
