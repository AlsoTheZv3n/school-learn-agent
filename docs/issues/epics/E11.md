## Zielbild

Das **Lehrer-Dashboard** ist die dichte, transparente Gegen-Sicht zur ruhigen Schülerseite (E10) — über **dieselben Daten** (`learner_state`), aber mit anderer Präsentation. Es macht drei nicht verhandelbare Prinzipien im UI sichtbar:

- **Open Learner Model (P5):** pro Skill `mastery` **und** `uncertainty` (+ `attempts_count`), damit die Lehrperson sieht, *warum* das System eine Einschätzung trifft und *wie verlässlich* sie ist.
- **Mensch im Loop (P6):** Notiz hinterlegen und optional Mastery überschreiben; die Notiz erscheint anschliessend auf der Schülerseite.
- **Safety in der DB (P1):** Die UI filtert **nicht** nach „eigenen Klassen" — RLS auf dem Backend erzwingt das. Kleine Kohorten liefern `403` → die UI zeigt „zu wenige Lernende für eine anonyme Auswertung" statt Zahlen.
