## Zielbild

E7 implementiert die **einzige Plugin-Naht** des Systems (Prinzip P7): eine fachgekeyte `GraderStrategy`-Registry. Eine Schülerantwort wird gegen einen **kuratierten** Answer Key bewertet (P2) und liefert ein `GradeResult(correct, feedback, confidence)`. Mathe wird symbolisch/numerisch und deterministisch geprüft (`confidence = 1.0`); Sprache regelbasiert; Geschichte als offene Antwort mit LLM-*Vorschlag* unter `confidence < 1.0` und Lehrer-Override-Pfad (P6).

Die Asymmetrie ist beabsichtigt (docs/00, Section 6): **Nur** `grading/` ist eine Plugin-Registry. `retrieval/`, `agent/`, `learner_model/` bleiben flache Module mit je einer Implementierung.
