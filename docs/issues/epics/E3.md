## Zielbild

Dieses Epic verankert die **Isolation zwischen Personen** dort, wo sie nicht umgangen werden kann: in der Datenbank. Es ist das "schmerzhaft nachzuruestende" Paket und wird deshalb frueh gebaut (P1). Zwei Leck-Flaechen werden geschlossen:

1. **Individual-Leak** — eine Query, die haette gescoped sein sollen, liefert fremde Schuelerzeilen. Gegenmassnahme: **Postgres Row-Level Security (RLS)** plus ein anwendungsseitiger **Scoping-Resolver**.
2. **Aggregat-Leak** — eine Population-Query ueber einen Filter, der genau eine Person trifft, wird zur de-anonymisierten Einzelauskunft. Gegenmassnahme: **Min-Cohort-Schwelle** (`k`, Default 10).

Beide Mechanismen sind **fail-closed**: im Zweifel 0 Zeilen bzw. Verweigerung, nie "alle". Der Erfolg des Epics ist messbar: Schueler A kann Zeilen von B nicht lesen, eine ungescopte Schueler-Session liefert 0 Zeilen, und eine Kohorte < `k` wird verweigert — bewiesen durch CI-blockierende Tests.

## Tasks

- **SAF-1 — RLS-Policies (Schueler sieht nur eigene Zeilen):** Rollen, Grants und RLS-Policies als versionierte Alembic-Migration auf `attempts`, `learner_state`, `teacher_notes`, `enrollments`; fail-closed via `current_setting` + `NULLIF`; `scoped_session` um `app.current_teacher_id` ergaenzen.
- **SAF-2 — Scoping-Resolver:** `safety/scoping.py` stellt sicher, dass keine Individual-Query ohne aufgeloesten Scope laeuft; fehlender Scope wirft `ScopeError`.
- **SAF-3 — Min-Cohort-Schwelle fuer Aggregate:** `safety/cohort.py` mit `enforce_min_cohort` als einziger, zentraler Stelle, durch die jede Population-Query laeuft; Default `k=10` aus `settings`.
- **SAF-4 — Safety-Tests (CI-blockierend):** `tests/test_rls.py` + `tests/test_cohort_threshold.py` beweisen Isolation und Schwelle gegen echtes Postgres und blockieren den Merge bei Fehlschlag.

> Alle Tasks tragen das Label `safety-critical`. Die Checkbox-Liste mit den konkreten Issue-Nummern wird separat ergaenzt.
