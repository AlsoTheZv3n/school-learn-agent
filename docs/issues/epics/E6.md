## Epic E6 — Learner-Modell (BKT)

**Milestone:** M3 — Learning Engine
**Quelldokument:** [`docs/06-learner-model-and-grading.md`](docs/06-learner-model-and-grading.md) (Teil A — Learner-Modell)

### Zielbild

E6 baut die zweite ITS-Kernkomponente: das **Learner-Modell**. Es liefert eine **interpretierbare, inspizierbare Mastery-Schätzung pro (Schüler:in, Skill)** auf Basis von **Bayesian Knowledge Tracing (BKT)**. Eine einzelne Schülerantwort (korrekt/falsch) wird über einen **einzigen Schreibpfad** deterministisch in eine aktualisierte Mastery + Unsicherheit überführt. Diese Werte sind später vom Open Learner Model (Lehrer-Dashboard, P5) lesbar und durch eine Lehrperson überschreibbar (P6).

### Bezug zu den Prinzipien

- **P5 — Open Learner Model**: BKT (interpretierbar) vor DKT (neuronal). Die Lehrperson sieht *warum* das System ein Kind einschätzt, inkl. Unsicherheit.
- **P3 — Das Learner-Modell verbessert sich, nicht der Agent**: ausschließlich das Modell wird aktualisiert; das Agent-Verhalten ist eine Funktion dieses Modells. Genau ein Schreibpfad (`record_attempt`).
- **P7 — Genau eine Plugin-Naht (`grading`)**: `learner_model/` bleibt ein **flaches Modul** mit *einer* Implementierung — DKT ist ein späterer Swap, kein paralleler Plugin.

### Tasks

- **LM-1 — BKT-Kern**: Reine NumPy-/Python-Funktionen (`posterior`, `update`, `mastery_after`) plus `BKTParams` — die testbare Bayes-Mathematik ohne Seiteneffekte.
- **LM-2 — Tracing-Service**: `record_attempt(...)` als einziger legitimer Schreibpfad in `learner_state`; hält Mastery, Unsicherheit und Versuchszähler konsistent (P3).
- **LM-3 — DKT-Platzhalter**: interface-kompatibler Stub + Aktivierungs-Doku („erst aktivieren, wenn genügend Historie vorliegt UND BKT messbar limitiert ist").
