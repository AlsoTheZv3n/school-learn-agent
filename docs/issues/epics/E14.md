## Tasks

- **PROD-1 — Produktiver Ingestion-Pfad:** `scripts/import_production.py` — idempotenter Import von echtem Material über die reguläre CON-2-Pipeline und von Personen/Rostern über Pydantic-validierten Upsert, strikt vom Mock getrennt.
- **PROD-2 — Env-Toggle Mock/Prod + Guards (`safety-critical`):** `DATA_MODE` plus getrennte `DATABASE_URL`s als einziger Modus-Schalter; Guards im Code (Seeder/Reset nur bei `mock`, Import nur bei `prod`), damit Mock- und Echtdaten nie dieselbe DB teilen.
- **PROD-3 — Datenresidenz & Retention (`priority:critical`):** dokumentierte/konfigurierte CH/EU-Residenz für DB und LLM-Inferenz, Retention-/Löschkonzept pro Datenkategorie (gestützt auf CASCADE), AVV/No-Training-Anforderung bei externer LLM-Nutzung; rechtliche Angaben als gegen aktuelle Quellen zu prüfen markiert.
