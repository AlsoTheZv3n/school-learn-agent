## Zielbild

E10 liefert die **ruhige Schüler-Session** des ITS als React/TypeScript-Anwendung unter `apps/web`:
ein Konzept zur Zeit, ermutigender Ton, schonende Lernstand-Anzeige (gerundetes Prozent **ohne**
Unsicherheit). Die Schülerseite ist eine von „zwei Sichten auf eine Wahrheit" — das Gegenstück zum
dichten Lehrer-Dashboard (E11). Sie konsumiert die Endpoints aus E9 (`POST /student/turn`,
`GET /student/mastery`) über einen einzigen, typisierten API-Client.

## Tasks

- **FE-S1 — React/TS-Projekt-Setup:** Scaffold `apps/web` (Vite + React + TS), getrenntes Routing
  `/student` und `/teacher`, typisierter `api/client.ts` mit Auth-Header und `turn()`/`myMastery()`.
- **FE-S2 — Session-Screen:** `SessionScreen.tsx` + `TutorThread.tsx` + `MasteryBar.tsx`; ein
  Konzept zur Zeit, schonende Mastery-Anzeige, Lehrernotiz-Anzeige, Unterstufen-Konfigurationsfall.
- **FE-S3 — Helfer-Aktionen:** drei Buttons („Anders erklären", „Hinweis", „Wozu?") rufen den
  generativen `explain`/`hint`/`why`-Pfad — strikt getrennt vom Bewertungspfad.
